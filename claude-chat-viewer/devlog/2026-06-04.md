## 架构变更

- **原方案**：Electron 桌面壳（150MB 二进制）
- **新方案**：Express 本地服务器 + 浏览器
- **原因**：Electron 二进制在用户网络下载极慢（镜像 5 分钟/5MB）
- **新架构优点**：零额外下载、UI 完全一致、天然跨平台
- **影响文档**：tech-spec.md、CLAUDE.md、dev-plan.md 已全部同步更新

## 关键技术发现：Tailwind CSS v4 编译 BUG

**现象**：多个 Tailwind 类（`fixed`、`px-6`、`py-5`、`ml-14`）写入文件后，Vite 正确输出源码，但浏览器不渲染这些类的样式。

**影响**：反复尝试 Tailwind 类调整布局均无效，浪费大量时间排查。唯一可靠的方式是 **inline style**。

**临时方案**：本项目所有布局、尺寸、颜色相关样式一律使用 React inline style，不使用 Tailwind 类。Tailwind 仅保留 `@theme` 自定义颜色变量供参考，以及在已确认可用的简单场景（如 `flex`、`text-sm`、`overflow-hidden`）中谨慎使用。

**待调查**：后续需要排查是 Tailwind v4 配置问题、Vite 插件兼容问题、还是 PostCSS 处理环节的 bug。
