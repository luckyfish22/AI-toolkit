# Claude Chat Viewer — 需求文档

> 一款本地应用，用于浏览和管理 Claude Code 的聊天记录。要求只读访问，不修改任何 Claude Code 原始数据。

---

## 一、应用定位

用户使用 Claude Code 写代码时，会产生大量聊天记录存放在 `~/.claude/` 目录下。本应用提供一个图形化的浏览界面，让用户可以方便地回顾、搜索、收藏这些对话。

**核心原则**：只读不写。任何操作都不能修改 `~/.claude/` 目录下的文件。

---

## 二、数据来源

### 2.1 Claude Code 聊天数据

Claude Code 的所有聊天记录存储在用户主目录下的 `.claude/` 文件夹中：

| 数据 | 位置 | 说明 |
|------|------|------|
| 历史索引 | `~/.claude/history.jsonl` | 每行一条历史记录，记录用户的每一次提示 |
| 对话转录 | `~/.claude/projects/<项目名>/<会话ID>.jsonl` | 每行一个 JSON 对象，记录完整对话过程 |
| 项目记忆 | `~/.claude/projects/<项目名>/memory/*.md` | 项目相关的记忆文件 |

### 2.2 项目目录命名

Claude Code 会将项目的真实路径编码后作为目录名。例如：

- 真实路径：`D:\my_projects\blog_app`
- 编码目录名：`D--my-projects-blog-app`

编码规则：盘符 `X:\` → `X--`，路径分隔符 `\` 和 `_` 都替换为 `-`。

这个编码是**不可逆**的（`\` 和 `_` 都变成 `-`），还原项目名时需要通过 other 手段推断。

### 2.3 转录文件内容

每行一条 JSON，常见类型：

| 类型 | 含义 | 关键内容 |
|------|------|---------|
| `user` | 用户输入的消息 | 文本内容 |
| `assistant` | Claude 的回复 | 包含文本、思考过程、工具调用等块 |
| `system` | 系统事件 | 如命令执行、耗时统计等 |
| `ai-title` | AI 自动生成标题 | 会话的简短标题 |

`assistant` 类型的回复由多个「块」组成：
- **文本块**：Claude 的文字回复，包含 Markdown 格式
- **思考块**：Claude 的内部推理过程
- **工具调用块**：Claude 调用的工具（读取文件、执行命令、搜索等）

### 2.4 应用自有数据

收藏夹等用户创建的数据，存放在系统应用数据目录（如 Windows 的 `%APPDATA%`、macOS 的 `~/Library/Application Support/`），与 Claude Code 数据完全隔离。

---

## 三、功能需求

### F1 — 项目列表与浏览

左侧面板显示所有聊天过的项目列表。

- 自动扫描 `~/.claude/projects/` 下所有项目
- 每个项目显示**人类可读的名称**，而非编码后的目录名
- 项目按最近聊天时间排序，最近的在上面
- 点击项目展开该项目的所有会话列表
- 每个会话显示：
  - AI 自动生成的标题（如有），否则显示第一条用户消息的前几十个字
  - 最后聊天日期（今天显示时间，本周显示"x天前"，更早显示月/日）
- 选中会话高亮显示
- 面板底部显示项目总数

**项目名还原**：需要通过以下方式获取项目的真实名称：
1. 从历史索引中查找编码名对应的真实路径，取最后一级目录名
2. 如果找不到，从转录文件中读取 `cwd`（工作目录）字段
3. 如果还找不到，从编码名中尽量提取可读的部分

### F2 — 对话查看

主区域以聊天气泡形式展示对话内容。

- **用户消息**：靠右对齐，浅色背景气泡，圆角
- **AI 回复**：靠左对齐，白色背景，占满宽度
- 支持 **Markdown 渲染**：
  - 标题（多级）
  - 列表（有序/无序）
  - 表格
  - 引用块
  - 链接（点击新标签页打开）
- **代码块**：语法高亮，显示语言标签，支持一键复制
- 会话顶部显示：标题（大字加粗）、使用的 AI 模型名、消息总数
- 首次打开时显示引导提示：「选择对话开始浏览」

### F3 — 全文搜索

侧边栏顶部提供搜索框，支持在所有聊天记录中搜索关键词。

- 快捷键聚焦搜索框
- 输入关键词后实时搜索（带防抖，避免频繁触发）
- 搜索范围：所有项目的所有会话的用户消息和 AI 回复
- 搜索结果按项目分组显示
- 每条结果显示匹配类型（用户/AI）和内容片段
- 点击结果跳转到对应会话，并**高亮**匹配的消息
- 搜索结果面板可关闭（点击外部区域或清空搜索）

### F4 — 收藏夹系统

用户可以创建收藏夹，将任意项目的任意会话加入收藏，跨项目整理。

**收藏夹管理**：
- 创建收藏夹（支持嵌套子收藏夹）
- 重命名收藏夹
- 删除收藏夹（删除时确认，同时删除其子收藏夹）
- 所有收藏夹数据存储在应用自有目录，不修改 Claude Code 原始数据
- 操作后短暂提示反馈（如「已添加」）

**添加会话**：
- 在会话列表中，每个会话 hover 时显示操作按钮
- 点击后可选择添加到已有收藏夹，或快速新建收藏夹
- 同一个会话可以加入多个收藏夹

**浏览收藏夹**：
- 侧边栏顶部提供视图切换：按项目浏览 / 收藏夹
- 收藏夹视图中，每个收藏夹可展开查看内部的会话列表
- 点击会话可跳转到对话详情
- 支持从收藏夹中移除会话

### F5 — 项目信息查看

每个项目可查看附加信息：

- **项目记忆**：展示 Claude Code 为该项目保存的记忆内容（存储在 `memory/` 目录下的 Markdown 文件）
- **项目权限**：展示该项目的本地权限配置（允许/拒绝的操作列表）

### F6 — 项目快捷操作

- **复制项目路径**：一键复制项目的原始文件系统路径到剪贴板
- **置顶项目**：可将常用项目固定在项目列表顶部

## 四、界面与交互设计

### 4.1 整体布局

```
┌──────────────┬────────────────────────────┐
│   侧边栏     │         顶部栏              │
│   固定宽度   ├────────────────────────────┤
│             │                            │
│  搜索框     │        对话内容区           │
│  视图切换   │                            │
│  项目/收藏  │   用户消息 ▎               │
│  列表       │        ▎   AI 回复         │
│             │                            │
│             │                            │
└──────────────┴────────────────────────────┘
```

- 侧边栏固定宽度（约 360px），始终可见
- 主区域自适应剩余宽度，内容居中（最大阅读宽度约 860px）
- 最小窗口尺寸：900×600

### 4.2 配色 — 暖米色浅色主题

整体色调温暖、干净、低视觉噪音，类似阅读类应用的舒适感。

| 用途 | 颜色 | 说明 |
|------|------|------|
| 页面底色 | 暖白 | 最外层背景 |
| 侧边栏 | 浅米灰 | 与主区域区分 |
| 卡片/面板 | 纯白 | 内容区域 |
| 主文字 | 近黑 | 正文标题 |
| 次要文字 | 中灰 | 时间戳、辅助信息 |
| 强调色 | 暖棕橙 | 按钮、链接、选中态 |
| 用户消息气泡 | 暖米色 | 区别于白色背景 |
| 边框/分割线 | 浅灰 | 低视觉重量 |

### 4.3 字体

- 正文：系统默认无衬线字体（Windows：微软雅黑，macOS：苹方）
- 代码：等宽字体（Consolas / SF Mono / 通用 monospace）
- 时间戳：小一号，灰色

### 4.4 交互细节

- 展开/折叠：带过渡动画（约 150ms），箭头图标旋转
- 悬停效果：背景色轻微变化
- 弹出菜单/对话框：带半透明背景遮罩，点击遮罩关闭
- 操作反馈：右下角短暂提示，自动消失
- 加载状态：旋转动画指示器
- 搜索高亮：匹配消息用浅橙色背景标记
- 滚动条美化：细条、圆角

### 4.5 空状态

各种无数据场景需要有友好提示：
- 未选中会话：居中图标 + "选择对话开始浏览"
- 无搜索结果："无匹配"
- 无收藏夹："暂无收藏夹"
- 无记忆/权限："该项目暂无记忆" / "该项目无本地权限配置"
